Architecture

End-to-End RAG Pipeline

A true Retrieval-Augmented Generation pipeline powered by two IBM watsonx.ai Granite models working together in real-time.

🧩

Embedding — Retrieval Layer

Every vendor query is encoded into a high-dimensional vector using the multilingual Granite embedding model. This enables semantic search across 22+ Indian languages — Hindi, Marathi, Tamil, Telugu, Kannada and more.

ibm/granite-embedding-278m-multilingual
⚡

Generation — Answer Layer

Once relevant documents are retrieved from the knowledge base, Granite 4H Small generates a structured, actionable, and context-aware response tailored to the vendor's specific business, location, and language.

ibm/granite-4-h-small
📚

Knowledge Base

18 pre-loaded real-world documents covering PM SVANidhi, MSME Udyam, Mudra Yojana, FSSAI, PhonePe/Paytm UPI setup, Swiggy/Zomato listing, Google My Business, hyperlocal SEO, WhatsApp Business, and city-specific data for Pune, Mumbai, Chennai.

🔍

Cosine Similarity Search

Query vectors are compared against all embedded knowledge base documents using cosine similarity. Top-k most relevant documents are injected as context into the generation prompt — ensuring grounded, factual responses every time.
